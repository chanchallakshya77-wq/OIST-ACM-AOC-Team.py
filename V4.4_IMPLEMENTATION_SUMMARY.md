## 🔄 Major Architectural Changes

### 1. Complete Removal of Automated Form Filling

**Original (Deprecated):**

**V4.4 (Current):**

**Rationale:** Systemic risk review identified automation as triggering:
- Third-party firewall blockages
- Anti-bot mechanisms
- CAPTCHA constraints
- Legal ambiguity around automated portal interaction

---

### 2. Introduction of 5 Structural Rectifications

V4.4 formalizes **5 critical engineering fixes** not explicitly detailed in the original architecture:

#### **Rectification #1: Hardware Disconnect & WebGPU Failures**
- **Problem:** 2-4GB RAM devices crash with local LLMs
- **Solution:** Adaptive hardware sniffing + TEE fallback
- **Status:** ✅ Implemented in `src/core/hardware-sniffer.ts`

#### **Rectification #2: Cryptographic Brittleness**
- **Problem:** SHA-256 hash false-positives on formatting variants
- **Solution:** Semantic normalization + Jaro-Winkler matching
- **Status:** ✅ Implemented in `src/core/normalization.ts`

#### **Rectification #3: TED Algorithm Latency**
- **Problem:** Cubic O(n³) tree-edit-distance causes UI freezes
- **Solution:** Linear O(n) subtree keyword scanning
- **Status:** ⏳ Designed, not yet implemented

#### **Rectification #4: Manual Handoff Abandonment Gap**
- **Problem:** Removing automation increases user friction
- **Solution:** Floating assistive HUD with click-to-paste
- **Status:** ⏳ Designed, not yet implemented

#### **Rectification #5: Mobile Data Tax**
- **Problem:** Large payloads penalize prepaid data users
- **Solution:** Bloom-filter encoded queries + Brotli < 50KB
- **Status:** ⏳ Designed, not yet implemented
